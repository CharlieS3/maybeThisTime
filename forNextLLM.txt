## Stack
- Frontend: Next.js 16.2.6 (App Router, TypeScript, ESLint, no Tailwind, no src dir, import alias `@/*`), runs at **http://localhost:3001** (`"dev": "next dev -p 3001"`).

## Environment
- One `.env` at `Code/.env` (UTF-8). Contains `DATABASE_URL=postgres://postgres:<pw>@localhost:5432/my_app` and `DB_PASSWORD`.
- `migrate.ts` loads it via `path.resolve(__dirname, "../../../.env")` (anchored to file).
- Seeds load it via `path.resolve(process.cwd(), "../../.env")` (cwd-relative → must run from `packages/database`).
- NestJS loads it via ConfigModule (see app.module.ts above).

## Database schema (current)
```sql
profiles  (id UUID PK DEFAULT gen_random_uuid(), first_name TEXT NOT NULL)
breweries (id UUID PK DEFAULT gen_random_uuid(), name TEXT NOT NULL)
schema_migrations (filename TEXT PK, applied_at TIMESTAMPTZ NOT NULL DEFAULT now())
```
- Migration runner skips files already recorded in `schema_migrations`; never edit applied migration files — add new numbered ones (ALTER TABLE etc.).

## Conventions we settled on
- **DB = snake_case, API/frontend = camelCase.** Translation happens in service-layer SQL via aliases: `SELECT id, first_name AS "firstName" FROM profiles` (double quotes required).
- **Explicit column lists, no SELECT \*** (deliberate exposure of fields; security).
- Parameterized queries always (`$1`, values array).
- Clients never supply `id` — DB generates UUIDs. DTOs exclude id.
- DTO properties use `!` (e.g. `firstName!: string;`).
- Controller = HTTP only; Service = logic + SQL; only DatabaseService touches the pool.
- Next.js: routes mirror screens, not actions (`/profiles`, `/profiles/new`; delete will be a button, not a page). Server components fetch for read pages; `"use client"` only where interactivity is needed.
- INSERTs use `RETURNING id, ... AS "camelCase"` to send the created row back.

## Working endpoints / pages (all verified)
- GET  /profiles  → `[{ id, firstName }]`
- POST /profiles  → body `{ "firstName": "..." }`, returns created row
- GET  /breweries → `[{ id, name }]`
- http://localhost:3001/profiles (list), /profiles/new (create form, works end to end), /breweries (list)
